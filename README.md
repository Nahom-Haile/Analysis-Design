# Analysis-Design

In the projects for this course, the main problem I was solving was how to design and build an advising assistance program for ABC University’s Computer Science department. The program needed to read course information from a file, store that information in an appropriate data structure, print a full list of courses in alphanumeric order, allowing an advisor to search for a specific course to view its title and prerequisites. The goal was not only to make the program work, but also to choose a data structure that best supports the program’s requirements. 

I approached the problem by first breaking it down into smaller parts and working on it week by week. Before writing code, I created pseudocode for loading course data, validating the file format, creating course objects, and storing those objects in different data structures. I evaluated vectors, hash tables, and binary search trees to compare their strengths and weaknesses. This helped me understand why data structures are important. The structure used to store data can affect how efficiently a program can search, sort, insert, and print information. For example, a vector is simple to use, but searching through it can take longer as the amount of data grows. A hash table is fast for lookups, but it does not naturally keep data sorted. A binary search tree supports ordered traversal, which made it useful for printing courses in alphanumeric order. 

One roadblock I encountered was making sure the file data was validated correctly before storing it. The program needed to check that each line has at least one course number and course title, while also allowing some courses to have no prerequisites. Another challenge was validating that each prerequisite listed in the file also existed as a course. I overcame these issues by separating the logic into steps. First, read and store the course numbers, then create the course objects, and finally check prerequisites against the list of valid course numbers. Breaking the problem into smaller functions made the logic easier to understand, test, and debug. 

This project expanded my approach to software design because it showed me the importance of planning before coding. Instead of immediately writing the program, I first used pseudocode, runtime analysis, and data structure comparisons to make a design decision. This helped me think more like a software developer because I had to consider not just whether the program worked, but also whether the design was efficient and appropriate for the problem. Evaluating Big O runtime helped me understand how a program might perform as the amount of data grows. 

My work on this project also improved the way I write programs that are maintainable, readable, and adaptable. I focused on using clear function names, meaningful variable names, comments, and separate functions for different tasks such as loading data, searching for a course, printing the course list, and validating input. This made the program easier to follow and easier to update later. If the course file format changed or if another data structure needed to be used, the program would be easier to modify because the code was organized into logical sections. 

This course helped me apply non-coding development methodologies by using pseudocode and algorithmic design before implementation. It also helped me develop C++ code using algorithms and data structures to solve a basic programming problem. Finally, it strengthened my ability to evaluate complex data structures and choose the one that best fits a given problem. These skills are important because strong software design depends on understanding both the problem requirements and the data structures that can solve them efficiently.
